먼저 알아야 할 단어들

이 설명서에서 계속 나오는 단어들이야. 이것만 알면 나머지는 쉬워!

단어 뜻
SaaS
핵심 개념		 Software as a Service의 약자야.
인터넷에서 바로 쓸 수 있는 서비스라는 뜻이야.

예를 들면:
- 넷플릭스 = 영상 SaaS (매달 구독료 내고 영상 봄)
- 구글 독스 = 문서 SaaS (웹에서 바로 문서 작성)
- 노션 = 메모/관리 SaaS

앱을 따로 설치할 필요 없이, 웹사이트 접속 = 바로 사용!
이 프로젝트도 웹사이트에 들어가면 바로 쓸 수 있어.
뉴스레터 이메일로 보내는 뉴스 요약 모음집이야.

매일 아침에 카카오톡으로 오는 "오늘의 뉴스" 알지?
그거랑 비슷한데, 이메일로 오고, 특정 분야(여기서는 윤활유!)의
전문적인 뉴스만 모아서 보내주는 거야.
AI / 인공지능 ChatGPT 알지? 그런 똑똑한 컴퓨터 프로그램이야.

여기서는 AI가 3가지 역할을 해:
1. 뉴스를 읽고 "이건 중요해!"라고 점수 매기기
2. 뉴스 내용을 요약하고 분석하기
3. 예쁜 뉴스레터로 글쓰기

사람이 하면 몇 시간 걸릴 일을 AI는 몇 분 만에 해!
파이프라인 일이 순서대로 자동 진행되는 흐름이야.

마치 공장의 컨베이어 벨트처럼:
1단계 끝나면 자동으로 2단계 → 자동으로 3단계 → ...
사람이 중간에 개입할 필요 없이 쭉 흘러가!
MI Market Intelligence의 약자.
"시장 정보 분석"이라는 뜻이야.

쉽게 말하면 "경쟁사는 뭐 하고 있지?", "시장 흐름이 어떻지?"
같은 정보를 체계적으로 모아서 분석하는 거야.
API 프로그램끼리 대화하는 방법이야.

마치 식당 종업원처럼:
손님(화면)이 "뉴스레터 만들어줘!" 하면
종업원(API)이 주방(백엔드)에 전달하고
결과를 다시 손님에게 가져다주는 거야.
서버 24시간 켜져 있는 컴퓨터야.

우리 집 컴퓨터는 끄면 아무것도 못 하지?
서버는 항상 켜져 있어서 누구든 언제든 접속할 수 있어.
인터넷 어딘가에 있는 회사의 컴퓨터에서 돌아가!
데이터베이스
(줄여서 DB)		 정보를 체계적으로 저장하는 디지털 창고야.

엑셀 파일이랑 비슷한데, 훨씬 크고, 빠르고, 안전해.
뉴스레터 실행 기록, 설정값, 수신자 이메일 등을
여기에 깔끔하게 보관해둬!

이게 뭐하는 서비스야?

이건 SK엔무브라는 회사를 위해 만든 서비스야.

SK엔무브가 뭐 하는 회사야?
SK에너지에서 분리된 윤활유(엔진오일) 전문 회사야. "ZIC"이라는 엔진오일 브랜드가 바로 이 회사 제품이야! 한국뿐만 아니라 전 세계에서 윤활유를 팔고 있어.

이 회사의 직원들은 전 세계 6개 나라의 윤활유 시장 소식을 항상 파악해야 해.
"경쟁사가 새 제품 냈대", "이 나라에서 규제가 바뀌었대" 같은 정보들!

그래서 이 서비스가 하는 일은:

6개 나라의 윤활유 관련 뉴스를
AI가 자동으로 수집 → 분석 → 정리 → 이메일 발송
이 6개 나라의 뉴스를 매주 자동으로 모아!

왜 이걸 만들었어? (Before vs After)

원래는 사람이 직접 이 일을 했어. 그걸 자동화한 거야!

😫

이전 (수동)

  • 사람이 6개 나라 뉴스를 직접 검색
  • 중요한 기사를 하나하나 골라냄
  • 엑셀에 정리하고 번역
  • 이메일 형태로 직접 디자인
  • 수신자 이메일을 하나하나 입력해서 발송
  • 한 번 만드는 데 반나절~하루 소요
  • 사람마다 품질이 달라짐
🎉

이후 (자동화 SaaS)

  • AI가 6개 나라 뉴스를 자동 수집
  • AI가 중요도 점수를 매겨 자동 필터링
  • AI가 분석하고 한국어로 정리
  • 예쁜 이메일 템플릿 자동 생성
  • 설정된 수신자에게 자동 발송
  • 버튼 한 번이면 끝 (수 분 소요)
  • AI 검수까지 거쳐서 일관된 품질
쉬운 비유

옛날에는 매주 화요일마다 사원이 도서관 6곳을 돌면서 신문 읽고,
중요한 기사를 오려서, 예쁘게 붙이고, 복사해서 나눠줬어.
이제는 AI 비서가 이 모든 걸 10분 만에 해주는 거야!

전체 흐름 한눈에 보기

버튼 하나만 누르면 이 모든 과정이 자동으로 진행돼!
각 단계를 하나씩 자세히 설명해줄게.

🔍
1단계: 키워드 생성
Keyword Generator · Google Gemini AI 사용
AI가 "지금 윤활유 시장에서 중요한 키워드가 뭘까?" 하고 생각해서
검색어를 만들어내. 예를 들면: "윤활유 가격", "엔진오일 신제품", "기유 공급" 같은 것들!
💡 왜 이게 필요해?
사람이 "윤활유"만 검색하면 관련 없는 뉴스도 많이 나와. AI가 전문적인 검색어를 만들어줘서 정확한 뉴스를 찾을 수 있어!
▼
🌐
2단계: 뉴스 수집
Collector · 4개 분야 x 6개 나라 = 24개 동시 검색
1단계에서 만든 키워드로 인터넷에서 뉴스를 긁어와!
6개 나라 x 4개 분야를 동시에 검색해서 수백 개의 기사를 모아.
📋 4개 분야가 뭐야?
1. 거시경제 원유 가격, 기유(원재료) 공급 동향
2. 전방산업 자동차, 트럭, 선박, 건설장비 시장 (이런 데서 윤활유를 쓰거든!)
3. 경쟁사 다른 윤활유 회사들의 신제품, 가격, 프로모션
4. 윤활유 규제 각 나라의 윤활유 관련 법규, 규격 변경
▼
🧹
3단계: 정리 & 중복 제거
Merger · 같은 뉴스 걸러내기
같은 뉴스가 여러 곳에서 나올 수 있잖아?
"아, 이 기사 아까 본 것과 같은 내용이네" 하고 중복을 제거해.
▼
⭐
4단계: 점수 매기기
Scorer · Anthropic Claude AI 사용
AI가 각 뉴스를 읽고 0~5점으로 점수를 매겨.
"이 뉴스가 SK엔무브 윤활유 판매에 얼마나 관련 있는지"가 기준이야.
🎯 점수 기준 예시
5
경쟁사 새 제품 출시, 가격 변경
4
윤활유 규격/규제 변경
3
자동차/트럭 시장 변화
0
관련 없음 (연예, 부동산 등) → 제외!
▼
💡
5단계: 뉴스 요약 보강
Enricher · 기사 핵심 내용 요약
높은 점수를 받은 기사들의 핵심 내용을 AI가 더 자세히 요약하고,
분류 태그(윤활유동향, 경쟁사활동 등)를 붙여줘.
▼
📦
6단계: 비슷한 기사 묶기
Grouper · 같은 주제끼리 그룹화
"전기차 관련" 기사 3개, "원유 가격 관련" 기사 2개 이런 식으로
비슷한 주제의 기사끼리 묶어서 읽기 편하게 정리해.
▼
📝
7단계: 뉴스레터 작성
Writer · Claude AI가 전문 뉴스레터 작성
드디어 AI가 예쁜 이메일 형태의 뉴스레터를 작성해!
나라별로 따로 만들고, 3개 섹션으로 구성해:
📰 뉴스레터 구성
1. 📋 핵심 인사이트 - 이번 달의 가장 중요한 포인트 3~5개
2. 📰 섹터별 뉴스 - 분야별로 정리된 상세 뉴스
3. 🎯 전략 제언 - "이런 상황이니까 이렇게 대응하세요" 제안
▼
🧐
8단계: 품질 검사 (자기 교정!)
Auditor · Claude AI가 다시 검수 · 이 서비스의 가장 특별한 기능!
다른 AI가 "이거 잘 썼나?" 하고 6가지 기준으로 검사해!
🔎 검사하는 6가지 기준
1. 정보 정확성 - 숫자나 사실이 맞는지
2. 한국어 품질 - 어색한 번역체가 아닌 자연스러운 한국어인지
3. 톤 & 스타일 - 전문적이고 비즈니스에 맞는 어조인지
4. 완전성 - 3개 섹션이 다 있는지
5. 실행 가능성 - 전략 제언이 구체적이고 실용적인지
6. 군더더기 없음 - 쓸데없는 ESG/CSR 이야기가 없는지

쉬운 비유

학생(Writer AI)이 작문을 쓰면,
선생님(Auditor AI)이 채점하고 "여기 고쳐와!" 하는 거야.
학생이 고쳐오면 다시 채점... 최대 3번까지!

9단계: 이메일 발송!
Sender · Gmail API로 실제 이메일 발송

5

서비스 구조 (3가지 파트)

이 서비스는 크게 3가지 파트로 나뉘어져 있어. 레스토랑에 비유해볼게!

🎨
프론트엔드
= 식당 홀 (손님이 보는 곳)

사용자가 직접 보고 클릭하는 예쁜 화면이야.

• 대시보드에서 상태 확인
• "실행" 버튼 누르기
• 설정 변경하기
• 뉴스레터 미리보기
Next.js + Tailwind CSS
👨‍🍳
백엔드
= 주방 (실제로 일하는 곳)

눈에 보이지 않지만 진짜 일은 여기서 해.

• AI 파이프라인 실행
• 뉴스 수집 & 분석
• 뉴스레터 생성
• 이메일 발송
Python + FastAPI
📁
데이터베이스
= 창고 & 냉장고 (기록 보관소)

모든 정보를 안전하게 저장하는 곳이야.

• 실행 기록 (언제, 어떻게)
• 설정값 (수신자, 일정 등)
• 완성된 뉴스레터 HTML
• 에러 로그
Supabase (PostgreSQL)
레스토랑 비유 전체 정리

손님(사용자)이 식당 홀(프론트엔드)에 와서 "스테이크 하나요!"(실행 버튼) 하면,
종업원(API)이 주방(백엔드)에 주문을 전달하고,
요리사(AI들)가 냉장고(데이터베이스)에서 재료 꺼내서 요리하고,
완성된 요리(뉴스레터)를 종업원이 다시 손님에게 가져다주고,
동시에 배달부(이메일)가 다른 손님들에게도 배달해주는 거야!

📂 프론트엔드와 백엔드가 대화하는 방법 (API)

화면에서의 동작 백엔드에 보내는 요청 비유
"실행" 버튼 클릭 POST /api/runs
→ "새 뉴스레터 만들어줘!"		 주문서 제출
진행 상황 확인 GET /api/runs/{id}
→ "지금 어디까지 왔어?"		 "주문 언제 나와요?"
실시간 업데이트 GET /api/runs/{id}/events
→ 실시간 진행 스트리밍		 주방 CCTV 보기
설정 저장 PUT /api/settings
→ "수신자/일정 이렇게 바꿔줘"		 단골 메모 변경
뉴스레터 미리보기 GET /api/newsletters/{id}
→ "완성본 보여줘"		 요리 시식

이런 거 궁금할 수도 있어!

Q. 이거 왜 만든 거야?
SK엔무브 직원들이 전 세계 6개 나라의 윤활유 시장 동향을 파악해야 하는데, 매번 사람이 직접 6개 나라 뉴스를 찾아서 읽고 정리하기엔 너무 시간이 많이 들어. 그래서 AI가 이 모든 과정을 자동으로 해주는 서비스를 만든 거야! 매주 반나절 이상 걸리던 일이 버튼 한 번이면 끝나게 됐지.
Q. "SaaS"라고 부르는 이유가 뭐야? 그냥 웹사이트 아니야?
단순한 웹사이트는 정보를 보여주기만 해. 근데 SaaS는 실제로 일을 해줘!

일반 웹사이트 = 식당 메뉴판 (보기만 함)
SaaS = 실제 식당 (주문하면 요리해서 가져다줌)

이 서비스는 뉴스 수집, AI 분석, 이메일 발송까지 실제로 해주니까 "서비스(Service)"인 거야. 그리고 인터넷에서 바로 쓸 수 있으니까 "소프트웨어를 서비스로 제공(SaaS)"하는 거지!
Q. 돈 벌 수 있는 거야?
SaaS는 보통 구독료를 받는 비즈니스 모델이야. 넷플릭스처럼 매달 돈을 내고 쓰는 거지.

이 서비스도 다른 산업의 회사들에 팔 수 있어! 예를 들면:
• 제약 회사 → "글로벌 제약 시장 뉴스레터"
• 반도체 회사 → "반도체 시장 동향 뉴스레터"
• 식품 회사 → "글로벌 식품 시장 뉴스레터"

키워드와 평가 기준만 바꾸면 어떤 산업이든 적용 가능해!
Q. 뉴스는 어디서 가져와?
Tavily라는 AI 전용 검색 엔진으로 인터넷에서 자동 수집해!

작동 방식:
1. Gemini AI가 "윤활유 가격 변동", "엔진오일 신제품" 같은 키워드를 만듦
2. 그 키워드로 6개 나라의 뉴스 사이트, 업계 매체, SNS 등을 검색
3. 4개 분야(거시경제, 전방산업, 경쟁사, 규제) x 6개 나라 = 24개 검색을 동시에!
4. 수백 개의 기사가 모이면 AI가 중요도 순으로 점수를 매겨서 필터링

마치 24명의 기자가 동시에 취재하는 것 같은 거야!
Q. AI가 틀리면 어떡해?
그래서 "자기 교정 시스템(Self-Correcting Pipeline)"이 있어!

1. Writer AI가 뉴스레터를 작성하면
2. Auditor AI가 6가지 기준으로 채점해 (정확성, 한국어 품질, 톤, 완전성, 실용성, 군더더기)
3. 평균 점수가 3.5점 미만이면 "여기 이상해, 이거 고쳐!" 하고 피드백과 함께 돌려보내
4. Writer AI가 피드백을 반영해서 다시 작성
5. 이걸 최대 3번까지 반복!

마치 편집장이 기자의 원고를 검수하는 것처럼 여러 번 교정하는 거야. 이래서 일반적인 AI 서비스보다 훨씬 높은 품질이 나와!
Q. 이걸 만드는 데 뭐가 어려운 거야?
크게 4가지가 어려워:

1. 여러 AI 연결: Gemini → Tavily → Claude → Claude(검수) → Gmail까지 5개의 서로 다른 AI/서비스가 순서대로 작동해야 해. 하나라도 에러나면 전체가 멈추거든!

2. 6개 나라 동시 처리: 6개 나라 x 4개 분야 = 24개를 동시에 빠르게 처리하면서도 서버가 터지지 않게 관리해야 해.

3. 자기 교정 루프: "불합격이면 다시 쓰고, 합격이면 다음으로" 같은 조건부 흐름을 구현하는 게 상당히 복잡해.

4. 안정적인 서비스: 사용자가 멀리서도 웹으로 접속하고, 24시간 돌아가고, 데이터가 안전하게 저장되는 것까지 신경써야 해.
Q. 이거 쓰려면 프로그래밍 알아야 해?
전혀 아니야! 그게 바로 SaaS의 장점이야.

사용자는 그냥:
1. 웹사이트에 접속해서
2. 설정 페이지에서 수신자 이메일, 발송 주기 설정하고
3. "Start" 버튼 누르면 끝!

나머지는 전부 시스템이 알아서 해. 넷플릭스에서 영상 보는 데 서버 관리를 알 필요 없는 것처럼!
Q. 내 컴퓨터가 꺼져 있어도 돌아가?
응! 이게 클라우드의 장점이야.

이 서비스의 모든 부분은 인터넷 어딘가에 있는 회사들(Vercel, Render, Supabase)의 서버에서 돌아가. 내 컴퓨터, 내 노트북과는 전혀 상관없어. 그래서 매주 화요일 새벽에도 자동으로 뉴스레터를 만들어서 보내줄 수 있는 거야!
